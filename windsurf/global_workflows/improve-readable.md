เขียนให้อ่านเข้าใจง่าย 

=> Clarity First – เขียนให้เข้าใจง่าย ชัดเจนว่าทำอะไร
=> Single Responsibility – หนึ่ง unit / file / function ทำหน้าที่เดียว
=> Consistent Naming & Structure – ชื่อสื่อความหมาย จัดเรียง / indent / spacing ให้สม่ำเสมอ
=> DRY & KISS – อย่าซ้ำซ้อน เขียนเรียบง่าย
=> Document Intent – คอมเมนต์อธิบายเหตุผล ไม่ใช่สิ่งที่โค้ดทำอยู่แล้ว
